A website that provides a database of all redistricting data of a large variety of categories for California.

My Role

Redefine the website experience to make data more searchable and implement it

The Challenge

Since there were so many different categories and groupings of the redistricting data, the organization of the website is key to helping people find the data they need. As the sole web developer for Statewide Database, I had a lot of freedom to implement what I conceptualized in HTML, CSS, Javascript, and PHP. Since I was using Bootstrap to quickly implement the style of the numerous pages in the website, I designed a custom homepage, navigation bar, and footer to differentiate the site.

The Process - Navigation

For users who do not know exactly what file they are searching for, I utilized the header navigation bar and footer to provide two different ways to navigate the website. I organized the data by years subcategorized by type of data in a nested mega navigation bar. In the footer, I selected the most popular categories to display all the years available for a different organization.

The Process - Search

For users who DO know exactly what file they are looking for, I developed a categorical search engine connected to a MySQL database. I populated the database by crawling through the archives and parsing the file names to categorize them.

Reflections

The trickiest part of this project was the amount of research I had to do to fully understand the best way to categorize redistricting data most intuitively. It was also a challenge to implement the multi-level mega navigation bar. Even though the final website is not yet deployed to the official site at statewidedatabase.org, I overall had a lot of fun doing this project.
