Spetcial is an early stage startup looking to transform the pet industry by providing an all-inclusive app for pet owners. The app had to be flexible enough to be a social media platform and be a database for all products, services, and trails/park related to pets.
Visit the website at spetcial.com.

My Role

As the first designer on the team, I collaborated with the product manager to conceptualize all of the features of the application from scratch. I helped come up with the product’s branding and set the foundations of its user interface.

Mobile Profile UX

Problem

How do you present a lot of information for a user’s profile in a limited amount of space?

Features

After discussion with the product manager and founder, these are the list of features that they wanted me to bring together:

  • Basic information about the pet and owner
  • A detailed “About” page to display the story of the pet and owner
  • Action items associated with the profile such as messaging, following, seeing the badges that the pet has earned
  • Photo, trails, services, and product filter categories for profile posts
  • A way to add new content

Research

Since I was starting from scratch, I had no idea where to start. I researched how other designers have solved similar problems in the past by browsing Pinterest, Creative Bloq, and the apps that I often use in my phone.

Design Concept

I fleshed out my ideas by heavily drawing inspiration from the layout of Facebook’s profile UI and Instagram’s double navigation UI. I had to go through a couple of iterations to figure out the best way to display the profile information.

Iteration 1

  • I decided to include the owner’s profile picture next to the main pet profile picture mainly to help personalize the pet profile more. Also, an owner profile picture is necessary for a future feature where owners will be matched to other owners for pet hangouts.
  • Since I was only allowed to use half of the screen for profile information, I needed to cram everything to the right of the profile picture. I designed icons for each item to decrease the amount of text as much as possible.

Iteration 2

  • While adding a new feature to allow for direct messaging of a user from the profile, I realized that grouping the action items (about, message, follow) together would help users find them more intuitively.
  • After discussing with the PM the need for gamification, we decided on badges. I added a display of the highest badge a user has earned next to the profile picture so that the badge would visually be associated with the pet.
  • When I asked my friends for feedback on the profile UI that I mocked up on Illustrator, they found it to be confusing and too overwhelming.

Iteration 3

  • I decided to do a complete UI redesign because the layout was too cluttered. To avoid having layers of buttons one after the other, I placed the profile picture in the middle of the profile and message buttons. I decreased the space that the extra details was taking up by putting them in a paragraph format.
  • To make it more obvious that the follower information, treats, and badge level are clickable buttons, I placed them in squares with plenty of padding to decrease the cluttered look.

Iteration 4

  • After surveying target users, we found that they were not as interested in how many treats (“likes”) a pet has received, so I removed it to further decrease the clutter.
  • I added labels to the action item buttons to be clarify what they are, but put the text and circle outline in gray to add the illusion of more space.
  • To allow users to personalize their profile more, I included an option to add a cover photo background.

First Time UX for Web

Problem

How do you define the first time user experience to encourage users to keep coming back?

Hypothesis

  • Signing up should be hassle-free and take less than 5 minutes so that the users can experience the product first before committing a lot of time for set up.
  • Users should be able to discover new content that is relevant to them each time they visit the website.
  • Adding awards can further incentivize users to take certain actions.

Features

  • Social media sign up options to fill in basic information fast
  • Allow profiles to have incomplete information
  • Introduce gamification with a badging system
  • Home page has new discoverable content that is relevant to the user

Prototyping

I sketched quick wireframes to help envision what sequence of events would happen when a user first signs up on the website.

  • When the user is not signed into the website, I envisioned the homepage to look something like Yelp since the product is aiming to become the database for everything pet-related. New content can be discovered with site wide trending items and featured items near a location.
  • Only the most basic information is required to sign up, so the profile will display a message that the profile is incomplete and that a badge will be rewarded for filling in information.
  • Once the user clicks into the message or visits his/her profile, there will be a highlighted message box at the top that can be dismissed that will encourage the user to fill in certain profile information. Filling out the profile is broken into parts by awarding different levels of a badge so that the user is not overwhelmed with filling out everything at once.
  • To make filling out information very easy, the items can be edited in line like LinkedIn.

Iteration 1

I fleshed out what the homepage would look like for non signed in users and signed in users based on my wireframe. Before I was able to get to designing mockups on Illustrator for the badges, this FTUE project was put on hold because the PM deemed that it was too advanced at the time for a product that was not even launched yet.

Iteration 2

Realizing that I was thinking too far ahead into the future, I took a couple steps back and made mockups for how the sign up process should look like for a very easy to implement FTUE.

  • By requiring only the most basic information and letting the rest of the fields be optional, I tried to make the sign up process as low commitment as possible.
  • Allowing for sign up with different social media will allow the user to sign up even faster.
Branding

Problem

How do you define the brand image for a pet application?

Research

Major pet brands such as Petco and Petsmart tended to follow a more rounded, friendly look similar to baby brands.

Hypothesis

A friendly but professional minimalistic logo appeals to both genders.

Goals

  • Target users: 70% female, 30% male
  • Enough economic stability to have a pet
  • 20-40 year olds

A friendly but professional minimalistic logo appeals to both genders.

Variations

  • First, I came up with a couple variations using different colors. I tried to be more inclusive of both cat and dog owners by creating a version with pointy ears that could either be a dog or a cat.
  • By showing only half of the dog’s face, I tried to make the logo more abstract and simple. Also, I worked towards adding more professionalism by paying special attention to symmetry and regularity.

Final Logo

  • After surveying 100 target users, we found that the mouthless dog with the blue color was the most well-liked.
  • It was then a challenge to make the finalized logo versatile to different colored backgrounds. Converting all the colored parts to white was not an option because the dog would take on the color of the colored background and change the entire feel of the logo. In the end, I made the background circle a lighter opacity and filled the dog with a white color.
Conclusion

Conceptualizing this product from scratch was a very enjoyable yet challenging process for me. My main regret for this project is that I could have conducted more user research to experiment with more unconventional UI ideas.
